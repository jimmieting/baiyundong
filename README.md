# 白云洞登山局小程序 V1.0

> 核心哲学：向上的秩序 (Order Upwards)

## 项目结构

```
baiyundong/
├── app.js                 # 主程序 - 状态机、地理围栏配置
├── app.json               # 全局配置
├── app.wxss               # 全局样式 - 黛青主题
├── custom-tab-bar/        # 自定义底部导航栏
├── pages/
│   ├── index/            # 攀登页面（核心）
│   ├── leaderboard/      # 巅峰排行榜
│   ├── cobuild/          # 共建页面
│   ├── history/          # 历史记录
│   └── recordDetail/     # 记录详情
├── cloudfunctions/
│   └── getOpenId/        # 获取用户OpenID
└── images/               # 静态资源
```

## 核心功能

### V1.0 已实现

1. **地理围栏系统**
   - 起点：埠兴村登山口 (26.070797, 119.372559)
   - 终点：白云洞主洞平台 (26.075214, 119.389145)
   - 50米强制校验

2. **状态机**
   - IDLE → RUNNING → ARRIVED → COMPLETED
   - 云端状态持久化，支持断网恢复

3. **身份系统**
   - ANONYMOUS（匿名态）
   - HONOR（荣誉态）
   - 冷启动原则：确权后历史记录不可追溯入榜

4. **排行榜**
   - 今日榜 / 本月榜 / 历史榜

5. **视觉规范**
   - 黛青背景 (#003344)
   - C位大圆按钮
   - 毛玻璃卡片
   - 呼吸灯效果

## 数据库集合

需要手动在云开发控制台创建以下集合：

1. **t_workout** - 打卡记录
2. **t_user** - 用户表
3. **t_culture** - 文案库
4. **t_feedback** - 反馈表
5. **t_cobuilder** - 共建者名录

详见 `docs/database-schema.md`

## 开发指南

1. 在微信开发者工具中打开项目
2. 修改 `app.js` 中的云环境ID
3. 部署云函数：`getOpenId`
4. 创建数据库集合
5. 编译预览
